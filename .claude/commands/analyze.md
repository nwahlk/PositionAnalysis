分析用户最新的持仓数据，生成 Markdown + HTML 报告。

## 执行步骤

1. 在项目根目录 `D:\workspace\positionAnalysis` 下运行：
   ```
   pa analyze --no-open
   ```

2. 等待命令完成，检查输出确认：
   - 解析到的数据日期正确
   - 基金和股票数量与预期一致
   - 总资产、总盈亏数值合理
   - 报告文件路径输出成功（`.md` 和 `.html` 各一份）

3. 读取生成的 Markdown 报告（`.md` 文件），向用户展示：
   - **投资建议摘要**：总体评价 + 具体操作建议
   - **需要关注的标的**：卖出/加仓建议及其理由
   - **风险提示**：集中度、波动率等警告

4. 如果命令执行出错：
   - 解析错误 → 检查 Obsidian 中的持仓 Markdown 文件格式是否符合 `## 基金` / `## 股票` 等章节标题 + 管道表格格式
   - 网络错误（股票 API 代理问题）→ 基金数据正常就继续，告知用户股票数据因网络问题缺失
   - 其他错误 → 读取错误信息，尝试修复后重试一次

## 配置

- 持仓数据目录：`E:\连坤的知识库\01-Projects 当前进行中的项目\投资建议`
- 报告输出目录：同上（Obsidian vault 内）
- 持仓文件命名：`持仓记录_YYYY-MM-DD.md`，工具自动取最新日期的文件

## 注意

- 报告中的买入/卖出建议基于排名、盈亏、夏普比率、回撤、波动率等多因子评分，仅供参考
- 股票数据可能因代理问题获取失败，基金数据通常正常
- 港股代码以 H 开头（如 H03690 美团-W），已做特殊处理
